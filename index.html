    Mike’s Comments #

    Hello, I’m Mike, the creator of Etcha. Throughout the docs, you may see blue comments like this, it’s just me talking about how I use Etcha or adding more details about something.

    I built Etcha to solve a problem around managing systems in an immutable, distributed, and consistent way. I needed a tool that can build the same image across multiple cloud providers, version the images, and seamlessly switch between them. Along with images, I needed a way to define a runtime configuration that may change periodically and allow for things to be removed that are no longer needed.

    Etcha solves these problems by providing a single CLI application for building images and runtime configurations. These artifacts are designed to be flexible and stored anywhere, giving folks the freedom to customize and integrate Etcha within their existing artifact repositories. I hope you find Etcha useful for your needs, and I appreciate any feedback you have on it, the docs, or anything else.
